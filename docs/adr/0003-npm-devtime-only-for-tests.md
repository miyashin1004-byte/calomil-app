# テストのためだけにnpm/package.jsonを導入する(本番はビルド不要のまま)

ロジック部分(目標カロリー計算・データ層など)をTDDで作るには自動テストの実行環境が要る。一方このアプリは `index.html` を直接開くだけで動く、ビルド不要・依存ゼロの静的サイトという性質を保ちたい。

`package.json` と軽量テストランナー(Vitest想定)を開発時の依存として追加する。本番のデプロイ物(`index.html`/`app.js`/`style.css`等)や配布方法(GitHub Pagesにそのまま置く)は変えず、`npm test` はローカルでの開発時にのみ使う。

依存ゼロを貫く(自作アサーションで代替する)選択肢もあったが、失敗箇所の特定や非同期処理の扱いを自前で作り込むコストの方が、開発時限定でnpmを使う手間より大きいと判断した。
